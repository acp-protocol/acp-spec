# ACP Discord Server

Welcome to the official Discord server for the AI Context Protocol (ACP)! This server is your hub for discussing the protocol, getting help with the CLI, sharing implementations, and contributing to the project.

## Channel Permissions

### New Members
- Read-only access to information channels initially
- Can view announcements and documentation
- Access to `#welcome` and `#getting-started`

### Verified Members
- Full access to all community and technical channels
- Can post in help channels and participate in discussions
- Access granted after completing onboarding

### Contributors
- Access to all contribution channels
- Can participate in RFC discussions
- Recognition for code contributions

### Maintainers
- Admin access to all channels
- Can manage server settings and moderation

## Getting Started

1. **Read the Welcome Message** - Check `#welcome` for server rules and quick links
2. **Complete Onboarding** - Follow the onboarding flow in `#getting-started`
3. **Select Your Role** - Choose roles that match your interests (Developer, AI Tool Creator, Contributor, etc.)
4. **Install the CLI** - Get started with ACP by installing the CLI tool
5. **Join the Discussion** - Introduce yourself in `#general` and ask questions in `#help`

## Community Guidelines

### Code of Conduct

We are committed to providing a welcoming and inclusive environment for all members. Please:

- **Be respectful** - Treat everyone with kindness and respect
- **Be helpful** - Share knowledge and help others learn
- **Be constructive** - Provide constructive feedback and suggestions
- **Follow the rules** - Adhere to Discord's Terms of Service and our server rules

### Help Channel Etiquette

When asking for help in `#help`:

1. **Search first** - Check if your question has been answered before
2. **Be specific** - Provide context, error messages, and code examples
3. **Use threads** - Keep discussions organized in threads
4. **Mark as solved** - React with ‚úÖ when your question is resolved

### Contribution Guidelines

When contributing:

1. **Read the docs** - Familiarize yourself with the [Contributing Guide](../../CONTRIBUTING.md)
2. **Follow RFC process** - Major changes go through the RFC process
3. **Link to GitHub** - Use GitHub for bug reports and feature requests
4. **Be patient** - Maintainers are volunteers and may take time to respond
