# Discord Server Moderation Guide

This guide outlines the moderation structure, roles, guidelines, and community management processes for the ACP Discord server.

## Moderation Structure

### Roles

#### Moderators
**Responsibilities:**
- Community management and oversight
- Enforcing code of conduct
- Resolving conflicts
- Managing channels and permissions
- Coordinating with maintainers

**Permissions:**
- Manage messages
- Manage channels
- Manage roles (limited)
- Kick/ban members (with approval)
- Access moderation logs

#### Helpers
**Responsibilities:**
- Technical support in `#help` channel
- Answering questions
- Providing guidance
- Escalating complex issues

**Permissions:**
- Manage messages in help channels
- Create threads
- Pin messages (in assigned channels)

#### Contributors
**Responsibilities:**
- Active code contributors
- RFC participation
- Documentation improvements
- Community engagement

**Permissions:**
- Access to contribution channels
- Post in `#rfc-discussions`
- Recognition for contributions

#### Maintainers
**Responsibilities:**
- Project maintainers
- Final decision authority
- Strategic direction
- Community leadership

**Permissions:**
- Full admin access
- All moderation capabilities
- Server configuration

## Guidelines

### Code of Conduct

We are committed to providing a welcoming and inclusive environment. All members must:

1. **Be Respectful**
   - Treat everyone with kindness and respect
   - No harassment, discrimination, or hate speech
   - Respect different opinions and experiences

2. **Be Helpful**
   - Share knowledge and help others learn
   - Provide constructive feedback
   - Answer questions when you can

3. **Be Constructive**
   - Provide helpful, actionable feedback
   - Focus on ideas, not individuals
   - Suggest improvements, not just criticism

4. **Follow the Rules**
   - Adhere to Discord's Terms of Service
   - Follow server-specific rules
   - Respect channel purposes

### Spam Prevention

**Prohibited:**
- Excessive self-promotion
- Repeated messages
- Off-topic content
- Unauthorized bots
- Link farming

**Action:**
- First offense: Warning and message deletion
- Second offense: Temporary mute (1-24 hours)
- Repeated offenses: Ban

### Help Channel Etiquette

**Guidelines:**
- Search before asking
- Be specific and provide context
- Use threads for follow-up questions
- Mark as solved when resolved
- Thank helpers

**Moderation:**
- Remove duplicate questions
- Archive solved threads after 7 days
- Escalate complex issues to maintainers

### Contribution Guidelines

**For Contributors:**
- Read [CONTRIBUTING.md](../../CONTRIBUTING.md) before contributing
- Follow RFC process for major changes
- Link to GitHub for bug reports
- Be patient with maintainer responses

**For Maintainers:**
- Review contributions promptly
- Provide constructive feedback
- Recognize contributions publicly
- Maintain clear communication

## Moderation Actions

### Warnings

**Issued for:**
- Minor rule violations
- First-time offenses
- Unintentional violations

**Process:**
1. Issue private warning via DM
2. Explain the violation
3. Provide guidance for improvement
4. Document in moderation log

### Mutes

**Issued for:**
- Repeated violations
- Spam or disruptive behavior
- Disrespectful conduct

**Duration:**
- First mute: 1-4 hours
- Second mute: 4-24 hours
- Third mute: 24-72 hours

**Process:**
1. Issue mute with explanation
2. Notify via DM
3. Document in moderation log
4. Review after mute expires

### Kicks

**Issued for:**
- Severe violations
- Persistent rule breaking
- Temporary removal needed

**Process:**
1. Document violations
2. Attempt to resolve via DM
3. Kick with explanation
4. Allow rejoin after review

### Bans

**Issued for:**
- Harassment or hate speech
- Malicious behavior
- Repeated severe violations
- Terms of Service violations

**Process:**
1. Document all violations
2. Consult with moderation team
3. Issue permanent ban
4. Document in moderation log
5. Report to Discord if necessary

## Channel-Specific Rules

### Information Channels

**`#welcome`, `#announcements`, `#roadmap`, `#documentation`**
- Read-only for new members
- Maintainers post announcements
- Keep on-topic
- No discussions (use other channels)

### Community Channels

**`#general`**
- Casual discussion
- Keep ACP-related
- Be respectful
- No spam or self-promotion

**`#showcase`**
- Share implementations
- Include code examples
- Explain use cases
- No advertising

**`#help`**
- Technical questions only
- Use threads
- Mark as solved
- Be patient

**`#cli-discussion`**
- CLI-specific topics
- Share tips and tricks
- Troubleshooting help
- Keep on-topic

**`#ideas`**
- Feature requests
- RFC discussions
- Constructive suggestions
- Link to GitHub when appropriate

### Technical Channels

**`#getting-started`, `#annotations`, `#constraints`, `#integrations`, `#language-support`**
- Technical discussions only
- Provide code examples
- Reference documentation
- Help others learn

### Contribution Channels

**`#contributing`, `#rfc-discussions`, `#bug-reports`, `#feature-requests`**
- Contributors and maintainers focus
- Follow contribution guidelines
- Link to GitHub
- Keep professional

## Conflict Resolution

### Process

1. **Identify the Issue**
   - Review messages and context
   - Understand all perspectives
   - Check moderation history

2. **Private Discussion**
   - DM involved parties
   - Listen to all sides
   - Seek clarification

3. **Mediation**
   - Facilitate discussion
   - Find common ground
   - Propose solutions

4. **Resolution**
   - Document agreement
   - Monitor follow-up
   - Escalate if needed

### Escalation

**When to Escalate:**
- Complex conflicts
- Policy questions
- Ban appeals
- Maintainer involvement needed

**How to Escalate:**
- Contact moderation team
- Provide full context
- Include relevant messages
- Suggest resolution

## Best Practices

### For Moderators

1. **Be Consistent**
   - Apply rules fairly
   - Document decisions
   - Follow procedures

2. **Be Transparent**
   - Explain actions
   - Provide context
   - Answer questions

3. **Be Professional**
   - Stay calm and respectful
   - Avoid personal opinions
   - Focus on community health

4. **Be Proactive**
   - Monitor channels regularly
   - Address issues early
   - Prevent problems

### For the Community

1. **Report Issues**
   - Use report feature
   - Provide context
   - Be patient

2. **Respect Decisions**
   - Trust moderation team
   - Appeal if needed
   - Learn from feedback

3. **Help Maintainers**
   - Follow guidelines
   - Self-moderate
   - Support others

## Appeals Process

### Ban Appeals

**Process:**
1. Submit appeal via DM to moderator
2. Explain circumstances
3. Acknowledge violations
4. Propose improvement plan

**Review:**
- Moderation team reviews
- Decision within 7 days
- Explanation provided
- Final decision is binding

### Warning Appeals

**Process:**
1. Discuss with issuing moderator
2. Provide additional context
3. Request review if needed

**Resolution:**
- Warning may be removed
- Explanation provided
- No further action if valid
